== Doc drift: 6 → 8 modalities (APPLIED 2026-04-05)

*Finding:* `deployment-modes.adoc` and `federation-readiness.adoc` both
refer to "six modalities" / "six modality stores (Graph, Vector, Tensor,
Semantic, Document, Temporal)". Authoritative source (CLAUDE.md, TeX
data model, `rust-core/` crate layout) confirms **eight**:
Graph, Vector, Tensor, Semantic, Document, Temporal, **Provenance**, **Spatial**.

*Applied:* see `git log` in verisimdb for fix commit.

*Files affected:*

* `docs/deployment-modes.adoc` — "six modalities" wording
* `docs/federation-readiness.adoc` — "six modality stores" + normalizer regeneration bullet list (6 → 8)

== Undocumented soundness requirement: aggregate-drift weight renormalisation (APPLIED 2026-04-05)

*Finding:* Def. `def:agg-drift` requires `Σ w_{m1,m2} = 1` across 28 pairs.
Per-octad absence is handled soundly by `d(⊥,·)=0` convention, but
aggregate-drift claims (e.g., `PROOF CONSISTENCY` thresholds) silently
change semantics between an octad with full shapes vs. an octad with
some shapes absent, unless weights are explicitly renormalised over
the present shapes.

*Applied:* Remark `rem:agg-drift-renorm` added to
`arcvix-octad-data-model.tex` §4.2 immediately after the
`def:agg-drift` definition, introducing per-federation weight
renormalisation `ŵ_{m1,m2} = w_{m1,m2} / Σ w_{m'1,m'2}` over
`{m'1,m'2} ⊆ modset'`. Notes equivalence with the "no-drift term"
formulation when combined with the existing `d(·,⊥)=0` convention.

*Files affected:*

* `arcvix-octad-data-model.tex` — new `\begin{remark}` block at §4.2

== Normalizer regeneration strategies need 8, not 6 (APPLIED 2026-04-05)

*Finding:* `federation-readiness.adoc` section "Normalizer Regeneration
Strategies Are Stubs" lists 6 regeneration strategies (Graph, Vector,
Tensor, Semantic, Document, Temporal). Real count is 8 — needs entries
for Provenance (re-construct hash chain from transformation history)
and Spatial (re-extract coordinates from document + graph geographic
references).

*Applied:* The primary list at lines 63-72 was already updated to 8
entries in a prior session (part of the "6→8 modalities" foldback).
This session updated the remaining stale references at lines 114
("Six modality-specific strategies needed" → "Eight") and 157
("all six modalities" → "all eight modalities" with explicit shape
enumeration). Federation-readiness doc is now internally consistent.

*Files affected:*

* `verisimdb/docs/federation-readiness.adoc` — lines 114, 157 updated
