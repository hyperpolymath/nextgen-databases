== Overview

This monorepo contains 5 database engines, each with its own query language.
Three are registered in the BoJ Server `database-mcp` cartridge with connection
management, query execution, and PanLL dashboard panels.

== BoJ Server Integration

=== database-mcp Cartridge

Located at `boj-server/cartridges/database-mcp/`:

**Adapter:** `adapter/database_adapter.v` (V-lang)

**FFI functions (C ABI via Zig):**

* `db_connect(backend_id) ‚Üí slot`
* `db_connect_verisimdb(url) ‚Üí slot`
* `db_connect_quandledb(url) ‚Üí slot`
* `db_connect_lithoglyph(url) ‚Üí slot`
* `db_connect_sqlite(path) ‚Üí slot`
* `db_disconnect(slot) ‚Üí result`
* `db_execute_sql(slot, query) ‚Üí result` ‚Äî PostgreSQL, SQLite
* `db_execute_vql(slot, query) ‚Üí result` ‚Äî VeriSimDB
* `db_execute_kql(slot, query) ‚Üí result` ‚Äî QuandleDB
* `db_execute_gql(slot, query) ‚Üí result` ‚Äî LithoGlyph
* `db_begin_query(slot) / db_end_query(slot)` ‚Äî Transaction boundaries
* `db_state(slot)` ‚Äî Connection state (disconnected/connected/querying/error)

== VQL Safety Levels (VQL-UT)

VeriSimDB's query language has a 10-level type-safe query pipeline:

[cols="^1,<3,<5", options="header"]
|===
| Level | Name | What It Prevents

| L1  | Construction Safety    | SQL/query injection
| L2  | Schema Pinning         | Schema drift between query and database
| L3  | Resource Linearity     | Connection/cursor leaks
| L4  | Session Protocols      | State machine violations (query before connect)
| L5  | Effect Tracking        | Unaudited side effects
| L6  | Scope Isolation        | Namespace boundary violations
| L7  | Information Flow       | Data lineage tracking failures
| L8  | Quantitative Bounds    | Resource exhaustion (unbounded queries)
| L9  | Proof Attachment       | Missing evidence for sensitive operations
| L10 | Cross-Cutting          | Composition bugs across query boundaries
|===

== Not Yet Integrated

* **NQC** ‚Äî Gleam-based Nested Query Calculus. Working implementation in `nqc/src/nqc.gleam`
  with Erlang FFI. Needs a `db_execute_nqc()` FFI function and database-mcp registration.
* **TypeQL-Experimental** ‚Äî Idris2 ABI definitions exist at `typeql-experimental/src/abi/`.
  Parser is stub-phase. Needs compiler work before integration.
